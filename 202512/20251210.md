> [!NOTE]
>
> #### **1. Nullish 병합 연산자 (??)**
>
> `null` 또는 `undefined` 일 때만 오른쪽 값을 사용하는 연산자.
>
> ```js
> const name = null;
> const result = name ?? "기본 이름";
> console.log(result); // "기본 이름"
> ```
>
> ➡️ 0, "", false는 **값이 있는 것**으로 판단하여 그대로 유지함.
>
> ---
>
> #### **2. 자바스크립트 에러 핸들링 개요 (try/catch)**
>
> ```js
> try {
>   const user = JSON.parse("잘못된 json");
> } catch (error) {
>   console.log("문제가 생겼습니다!", error);
> }
> ```
>
> - 프로그램이 에러로 멈추는 것을 방지
> - 네트워크/JSON 파싱 등 실패 가능성이 있는 코드에서 사용
>
> ---
>
> #### **3. JavaScript sort() 함수 — 문자열 정렬에 주의**
>
> - 기본 정렬은 **문자열 기준**
>
> ```js
> [10, 2, 5].sort();
> // ["10", "2", "5"]
> ```
>
> - 숫자 정렬하려면 비교 함수 필요.
>
> ```js
> [10, 2, 5].sort((a, b) => a - b);
> // [2, 5, 10]
> ```
>
> ➡️ sort()는 **원본 배열을 변경**함 → 불변성 깨짐.
>
> ---
>
> #### **4. 불변성(Immutability) — 원본 데이터를 직접 수정하지 않기**
>
> ```js
> const arr = [1, 2, 3];
> const copy = [...arr]; // 복사
> ```
>
> - 예측 가능한 코드 유지
> - React 상태 관리에서 매우 중요
>
> ---
>
> #### **5. 자바스크립트 변수 유효 범위: var, let, const**
>
> **var**
>
> - 함수 범위(function scope)
> - 의도치 않은 버그 발생 가능
> - 사용 지양
>
> **let**
>
> - 블록 범위(block scope)
> - 값 변경 가능
>
> **const**
>
> - 블록 범위(block scope)
> - 값 변경 불가(참조값 유지)
>
> ---
>
> #### **6. var와 let의 예상치 못한 차이 — 호이스팅**
>
> ```js
> console.log(a); // undefined
> var a = 10;
> ```
>
> ```js
> console.log(a); // ReferenceError
> let a = 10;
> ```
>
> ➡️ `let`은 호이스팅되지만 **TDZ(Temporal Dead Zone)** 때문에 사용 전 접근 불가.
>
> ---
>
> #### **7. 배열 복사와 정렬**
>
> - 배열 복사
>
> ```js
> const copy1 = [...arr];
> const copy2 = arr.slice();
> const copy3 = Array.from(arr);
> ```
>
> - 정렬 시 원본 보호
>
> ```js
> const sorted = [...arr].sort((a, b) => a - b);
> ```
>
> ---
>
> #### **8. 로컬 스토리지(LocalStorage) & 메모 관리**
>
> ```js
> localStorage.setItem("name", "Yeonhwa");
> localStorage.getItem("name"); // "Yeonhwa"
> localStorage.removeItem("name");
> ```
>
> - 문자열만 저장됨
> - 브라우저에 반영구적으로 저장됨(자동 삭제 X)
>
> ---
>
> #### **9. 자바스크립트 Truthy / Falsy + 드모르간 법칙**
>
> - Falsy 값
>
> ```
> false, 0, "", null, undefined, NaN
> ```
>
> - 드모르간 법칙
>
> ```
> !(A || B) === !A && !B
> !(A && B) === !A || !B
> ```
>
> 예시:
>
> ```js
> if (!(isLogin || isAdmin)) {
>   // 로그인도 관리자도 아님
> }
> ```
>
> ---
>
> #### **10. 자바스크립트 날짜(Date) 사용**
>
> ```js
> const now = new Date();
> now.getFullYear();
> now.getMonth() + 1; // (0부터 시작)
> now.getDate();
> ```
>
> - 날짜 조작
>
> ```js
> const tomorrow = new Date();
> tomorrow.setDate(tomorrow.getDate() + 1);
> ```
>
> ---
>
> #### **11. NaN — 숫자가 아닌 값(Not a Number)**
>
> ```js
> Number("안녕"); // NaN
> ```
>
> - NaN은 자기 자신과 같지 않음
>
> ```js
> NaN === NaN; // false
> ```
>
> - 올바른 NaN 체크
>
> ```js
> Number.isNaN(value);
> ```
>
> ---
>
> #### **12. 변수 타입 확인 방법**
>
> **typeof**
>
> ```js
> typeof "hi"; // "string"
> ```
>
> **Array 확인**
>
> ```js
> Array.isArray([]);
> ```
>
> **정밀 타입 확인**
>
> ```js
> Object.prototype.toString.call([]);
> // "[object Array]"
> ```
