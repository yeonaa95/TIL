>
> #### 1. JSON Method
>
> JSON은 **데이터를 주고받기 위한 포장 박스**
> JSON Method는 그 박스를 포장하거나 개봉하는 도구 두 개
>
> - **JSON.stringify()** <br>
>   ➡ 객체를 **택배 박스(문자열)** 로 포장
>
> ```js
> const user = { name: "연화", age: 20 };
> const box = JSON.stringify(user);
> ```
>
> - **JSON.parse()** <br>
>   ➡ 택배 박스를 열어서 다시 객체로 꺼내기
>
> ```js
> const unpacked = JSON.parse(box);
> ```
>
> ➡️ 문자 ↔ 객체 변환기 두 개!
>
> ---
>
> #### 2. JSON Dynamic — “상자 안 내용을 실시간으로 바꾸기”
>
> Dynamic(동적) = 상황에 따라 내용이 바뀜. <br>
> ➡ 즉, JSON 데이터를 필요할 때 **실시간으로 만들거나 수정** 하는 것.
>
> ➡ 예시
>
> ```js
> const user = {};
> user.name = "연화";
> user.age = 20;
>
> const json = JSON.stringify(user);
> ```
>
> ➡️ 빈 상자에 하나씩 물건 넣고 → 나중에 포장하는 느낌.
>
> ---
>
> #### 3. DOM Tree — “웹페이지의 가계도(가족 나무)”
>
> DOM(Document Object Model)은  
> 웹페이지의 모든 요소(태그)를 트리 구조로 표현한 것.
>
> ➡ 비유
> : HTML 문서는 **가족 구성도(트리)** 처럼 생김.
>
> - `<html>` → 최상위 부모
> - `<body>` → 자식
> - `<div>` → 또 그 자식
> - `<p>` → 또 자식
>
> ➡ 트리 모양 예시
>
> ```
> html
>  └── body
>       ├── header
>       └── section
>             ├── h1
>             └── p
> ```
>
> ➡️ 웹페이지는 부모–자식 구조!  
> : 이 구조 자체가 DOM Tree.
>
> ---
>
> #### 4. DOM Control (DOM 조작) — “HTML을 JS로 움직이기”
>
> DOM Control = DOM Tree에서  
> ➡ 요소를 **찾고, 바꾸고, 만들고, 삭제하는 작업** 전체.
>
> ➡ **HTML**
>
> ```html
> <p id="msg">안녕</p>
> ```
>
> ➡ **JS**
>
> ```js
> const p = document.getElementById("msg");
> p.textContent = "Hello!";
> ```
>
> ➡️ **웹페이지 내용을 JS로 수정** = DOM Control.
>
> ---
>
> #### 5. DOM Event — “웹페이지에서 발생하는 모든 사건”
>
> ➡ **Event = “웹에서 일어나는 모든 일”**
>
> - 클릭
> - 스크롤
> - 키보드 입력
> - 마우스 올림
> - 데이터 입력
>
> ➡ 예시
>
> ```js
> button.addEventListener("click", () => {
>   console.log("버튼 클릭됨!");
> });
> ```
>
> ➡️ “무슨 일이 일어났는가?”를 감지하는 기능.
>
> ---
>
> ### 6. DOM Event Control — “사건이 일어났을 때 반응을 정하기”
>
> Event Control =  
> ➡ 이벤트가 발생했을 때 **무엇을 할지 결정하는 것**.
>
> ➡ 예시
>
> ```js
> input.addEventListener("input", () => {
>   console.log("글자가 입력됨!");
> });
> ```
>
> ➡️ “사건이 일어나면 이렇게 반응해!”
>
> ---
>
> ### 7. DOM Event Delegation — “아이들 사건을 부모가 대신 처리하기”
>
> **비유(초간단)**  
> 유치원 선생님이 아이들 30명에게 각각 말하는 대신  
> “누구든 손들면 선생님이 처리할게!”  
> → 부모 요소 한 곳에서 모든 자식 이벤트 처리.
>
> ➡ 왜 필요?
>
> - 자식 요소가 매우 많을 때
> - 자식이 계속 생기고 없어질 때
>
> ➡ **HTML**
>
> ```html
> <ul id="menu">
>   <li>사과</li>
>   <li>바나나</li>
>   <li>포도</li>
> </ul>
> ```
>
> ➡ **JS**
>
> ```js
> const menu = document.getElementById("menu");
>
> menu.addEventListener("click", (event) => {
>   if (event.target.tagName === "LI") {
>     console.log(event.target.textContent + " 클릭됨!");
>   }
> });
> ```
>
> ➡️ `<li>` 마다 이벤트를 달지 않고  
> ➡️ `<ul>` 한 곳에서 전체 클릭을 처리!
